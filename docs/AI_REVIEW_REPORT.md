## 3. 解消済みの前回懸念

### 3.1 分野表示指標と交渉ポイント参照指標の同一分野チェック

`validate:data` / `data-integrity.test.ts` で、`NEGOTIATION_POINTS.indicatorIds` が同一 `fieldId` の `relatedIndicatorIds` に含まれることを検証するようになった。前回の「別分野の指標を説明ポイントだけが参照する」問題は解消済み。

### 3.2 印刷シートの出典識別

印刷シートの出典一覧は、`SourceSummary.label` と `seriesCode` を表示するようになり、BOJ 系列を個別に追えるようになった。`/sources` でも系列コードと原データ CSV リンクを確認できる。

### 3.3 CGPI / SPPI の区別

`docs/DATA_CATALOG.md` と `docs/DATA_UPDATE.md` に、CGPI は `db=PR01` / `PRCG20_`、SPPI は `db=PR02` / `PRCS20_` と明記された。`sppi-industrial-waste-disposal` も SPPI として正しく扱われている。

## 4. 残る懸念点

### 4.1 追加指標が交渉ポイントに十分反映されていない

分野の `relatedIndicatorIds` には入っているが、交渉ポイントの `indicatorIds` には出てこない指標がある。バリデーション上は問題ないが、利用者から見ると「カードにはあるが、説明ポイントでは使われない」状態になる。

特に見直したいもの:

| 分野 | 指標 | 現状 |
|---|---|---|
| めっき | `cgpi-water` | 交渉ポイント本文は「用水コスト」と書くが、関連指標欄には出ない |
| めっき | `sppi-industrial-waste-disposal` | 排水処理費を補足資料に出すが、産廃処理指標は関連指標欄に出ない |
| 熱処理 | `cgpi-lpg` | 指標カードは出るが、炉エネルギー説明では触れない |
| 熱処理 | `cgpi-heavy-oil-a` | 指標カードは出るが、炉エネルギー説明では触れない |
| 板金 | `cgpi-bolts-nuts` | 指標カードは出るが、購入部品・副資材の説明ポイントがない |
| 研磨 | `sppi-industrial-waste-disposal` | 指標カードは出るが、研磨粉・廃液処理費の説明ポイントがない |

推奨対応:

1. 既存の交渉ポイントに `indicatorIds` と本文を追加する。
2. もしくは、用途が限定的なものは新しい交渉ポイントを追加する。
3. 使わない指標は分野マッピングから外すか、「補助指標」として位置づけを明記する。

### 4.2 ドキュメントに古い記述が残っている

`docs/DATA_UPDATE.md` にはまだ「v1.1 で 18 指標に拡充済み」という記述が残っている。現状は 28 指標なので、公開前に更新したい。

また、本レポートは今回更新済みだが、レビュー報告書を履歴として残す運用なら「過去レビュー」と「現行レビュー」の区別を明記すると誤読を防げる。

### 4.3 分野ページ下部の出典概要はまだ識別しづらい

`/sources` と印刷シートは改善済みだが、分野ページ下部の「出典概要」は `日本銀行 / 企業物価指数（2020年基準）` が並ぶ表示のままで、指標名・系列コードを出していない。

致命的ではないが、ページ内だけで根拠を追いやすくするなら、印刷シートと同様に `label` と `seriesCode` を表示するとよい。

## 5. 推奨する次アクション
