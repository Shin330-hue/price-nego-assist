# 設計書

**プロジェクト名:** 取適法・公的データ対応 価格協議準備ツール（金属加工業向け）  
**文書バージョン:** 2.0.0  
**作成日:** 2026-06-26  
**対象読者:** 開発者、CLI型AIコーディングエージェント  
**前提:** v1.0は静的フロントエンドのみ。個人情報・営業秘密・APIキーを扱わない。

---

## 1. 設計方針

### 1.1 コアコンセプト

本ツールは、金属加工業の中小受託事業者が、取引先へ価格協議を申し入れるための準備を支援する公開Webアプリである。アプリの中核は「グラフを見ること」ではなく、**公的根拠に基づく価格協議準備シートを作ること**である。

### 1.2 主要な設計判断

| 判断 | 内容 | 理由 |
|---|---|---|
| 静的SPA | バックエンド・DBなしで公開する | 利用ハードルと運用コストを下げるため |
| 公的データはJSON管理 | 実行時API取得ではなく、監査可能なJSONとして管理する | 出典確認、CORS回避、表示の安定性確保 |
| 法令情報は専用データ型 | `LegalBasis` として法令・指針情報を一元管理する | 条文番号ミスや表現の散在を防ぐため |
| ダッシュボードと印刷シートを分離 | 通常画面は閲覧性、印刷画面は提出資料性を重視する | PDF化したときの実用性を高めるため |
| AI生成はv1では入れない | 依頼文は監修済みテンプレートで対応する | 法務・営業リスクと実装複雑性を抑えるため |
| 自社データは保存しない | ユーザー固有データは空欄・チェックリストで扱う | 営業秘密・個人情報を扱わないため |

### 3.1 不正な `fieldId` の扱い

- `fieldId` が `fields.ts` に存在しない場合、`NotFoundPage` を表示する
- トップページへの導線を出す
- 自動リダイレクトより、ユーザーに「存在しない分野です」と伝える設計を推奨

### 3.2 SEO/OGP

- SPAであっても基本メタタグは `index.html` に設定する
- 可能であれば `react-helmet-async` 等でページタイトルを切り替える
- X共有向けに `og:title`、`og:description`、`og:image`、`twitter:card` を設定する

## 6. 計算設計

### 6.1 変化率の種類

| 表示名 | 計算 | 使用条件 |
|---|---|---|
| 基準年比 | `(latest - 100) / 100 * 100` | `baseYear` があり、指数値の基準が100の場合 |
| 表示期間内変化率 | `(latest - first) / first * 100` | データ配列の先頭から最新値の変化を表示する場合 |
| 前年比 | `(latest - prior) / prior * 100` | frequencyに応じた前年データが存在する場合 |

### 6.2 前年比の比較先

| frequency | 比較先 |
|---|---|
| monthly | 12件前、または同じ月の前年データ |
| quarterly | 4件前、または同じ四半期の前年データ |
| annual | 1件前、または前年データ |

単純な「5データポイント前」比較は禁止する。

### 6.3 計算ユーティリティ

```ts
export function getLatestPoint(values: CostDataPoint[]): CostDataPoint | null;
export function getFirstPoint(values: CostDataPoint[]): CostDataPoint | null;
export function calculateBaseYearChange(indicator: OfficialCostIndicator): number | null;
export function calculateDisplayPeriodChange(indicator: OfficialCostIndicator): number | null;
export function calculateYoYChange(indicator: OfficialCostIndicator): number | null;
export function formatRate(value: number | null): string;
export function formatValue(value: number, unit: string): string;
```

### 6.4 ラベル設計

- `calculateBaseYearChange` の結果だけを「2020年平均比」などと表示する
- `calculateDisplayPeriodChange` の結果は「表示期間内」または「2022年1月比」等、具体的に表示する
- `calculateYoYChange` が `null` の場合、前年比欄は `—` とする

## 10. 法令データ設計

### 10.1 管理場所

`src/data/legal-bases.ts` に定義する。将来的にJSON化してもよい。

### 10.2 データの作成ルール

- 条文番号は公式ソースで確認する
- `plainSummary` は平易な要約にする
- `negotiationMeaning` はユーザーがどう行動すべきかに結びつける
- `caution` は個別判断を避ける注意文を含める
- ソースURLと確認日を必須にする

### 10.3 v1.0の推奨法令カード

- 取適法の目的・位置づけ
- 発注内容等の明示義務（第4条）
- 代金の減額の禁止（第5条第1項第3号）
- 買いたたきの禁止（第5条第1項第5号）
- 協議に応じない一方的な代金決定の禁止（第5条第2項第4号）
- 報復措置の禁止（第5条第1項第7号）
- 価格交渉の記録作成（労務費転嫁指針）

### 11.2 テンプレート例

## 17. 追補（v1.1）：信頼区分UI・出典リンク・データ拡充の設計

### 17.1 出典リンクの常時表示

- `SourceBadge` に任意の `sourceUrl` を追加し、与えられた場合は `publisher / sourceName` を控えめな `ExternalLink`（別タブ・`noopener noreferrer`）として表示する。未指定時はテキスト表示で後方互換。
- `CostIndicatorCard` は各指標カードに常時見える出典リンクを表示する（折りたたみの詳細出典は維持）。

### 17.2 信頼区分のUI設計

- データ（指標カード・グラフ）には**肯定的な信頼キュー**（「出典記載の公的実データ」）を添える。
- 進め方ガイド・依頼文テンプレートには**「参考」バッジ＋注記**を付け、データと視覚的に分離する。印刷シートの注意書きにも「進め方・文面は参考」を含める。

### 17.3 データローダのスケール化

- 指標JSONの増加に備え、`data-loader` は `import.meta.glob('../data/official-indices/*.json', { eager: true })` で全件を自動集約する（id昇順で安定化）。`data-validation`・`validate-data` は `OFFICIAL_INDICATORS` 経由で自動追従する。

### 17.4 指標の出典タイプ拡張

- 出典は CGPI（日銀）・最低賃金（厚労省）に加え、**貿易統計（財務省）**を扱う。`OfficialCostIndicator` の型変更は原則不要（`publisher`/`sourceName`/`unit` で表現）。必要なら後方互換の任意 `sourceType?` を追加してよい。
- 指数でない系列（輸入単価=円/kg 等）は `baseYear` を持たず、基準年比は算出せず表示期間内変化率を主指標にする。

### 17.5 長期チャート

- 画面は軸ラベル間引き（`interval="preserveStartEnd"` + `minTickGap`）で約120点でも破綻させない。
- 印刷（`compact`）は点が多い場合に年平均へ間引いて可読性を確保する。
