## 6. 計算関数仕様

`src/lib/calculations.ts` に実装する。

### 6.1 `getLatestPoint`

```ts
export function getLatestPoint(values: CostDataPoint[]): CostDataPoint | null
```

- 空配列なら `null`
- 最後の要素を返す
- 呼び出し前に昇順ソート済みであることを前提にしてよいが、テストで順序を検証する

### 6.2 `calculateBaseYearChange`

```ts
export function calculateBaseYearChange(indicator: OfficialCostIndicator): number | null
```

- `baseYear` が未設定なら `null`
- 最新値が取得できない場合 `null`
- 計算式: `((latest.value - 100) / 100) * 100`
- 表示ラベル例: `2020年平均比`

### 6.3 `calculateDisplayPeriodChange`

```ts
export function calculateDisplayPeriodChange(indicator: OfficialCostIndicator): number | null
```

- 先頭値または最新値がない場合 `null`
- 先頭値が0なら `null`
- 計算式: `((latest.value - first.value) / first.value) * 100`
- 表示ラベル例: `表示期間内変化率` または `2022年1月比`

### 6.4 `calculateYoYChange`

```ts
export function calculateYoYChange(indicator: OfficialCostIndicator): number | null
```

- `monthly`: 最新periodの前年同月を探す。なければ12件前を使ってもよい
- `quarterly`: 最新periodの前年同四半期を探す。なければ4件前
- `annual`: 最新periodの前年を探す。なければ1件前
- 比較先がなければ `null`
- 比較先値が0なら `null`
- 「5データポイント前」固定比較は禁止

### 6.5 `formatRate`

```ts
export function formatRate(value: number | null, options?: { signed?: boolean }): string
```

- `null` は `—`
- 小数点1桁
- `signed: true` の場合、正値に `+` を付ける

### 6.6 `hasNonOfficialData`

```ts
export function hasNonOfficialData(indicators: OfficialCostIndicator[]): boolean
```

- `quality` が `official_verified` または `official_manual` 以外のものがあれば `true`

## 10. データ検証仕様

### 10.1 `validate-data.ts`

`scripts/validate-data.ts` は以下を検証する。

- すべての指標が必須項目を持つ
- すべての指標の `values` が空でない
- `period` が昇順である
- `value` が数値である
- `frequency` が正しい
- `quality` が正しい
- `sourceUrl` が `https://` で始まる
- `retrievedAt`、`publishedAt` がISO日付である
- `fields.relatedIndicatorIds` が存在する指標IDを参照している
- `negotiationPoints.indicatorIds` が存在する指標IDを参照している
- すべての `LegalBasis` に `sourceUrl`、`checkedAt` がある
- `NODE_ENV=production` または `VALIDATE_PRODUCTION=true` の場合、`quality: "sample"` が存在したら失敗

### 10.2 エラーメッセージ

エラーは修正しやすいように以下の形式で出す。

```txt
[data-validation] official-indices/cgpi-steel.json: missing sourceUrl
[data-validation] legal-bases.ts: unilateral-price-decision-prohibition missing checkedAt
[data-validation] fields.ts: cutting references unknown indicatorId cgpi-unknown
```

- 比較値が0の場合に `null`

## 15. 実装禁止事項

- フロントエンドにAPIキーを埋め込まない
- `VITE_FRONTEND_FORGE_API_KEY` 等のクライアント露出APIキーを作らない
- 法令条文番号を推測で記載しない
- 旧仕様の `第4条第1項第3号 代金の一方的決定の禁止` 等をそのまま移植しない
- 「必ず値上げできます」「違法です」と断定しない
- ユーザーの個人情報・取引先情報・金額情報を保存しない
- sampleデータを公式データのように見せない
- `getYoYChange` を5データポイント前固定で実装しない
- ダッシュボード全体だけを印刷対象にしない。印刷用シートを作る

## 18. 追補（v1.1）：追加文言・データ取得レシピ・指標カタログ

### 18.1 追加表示文言

- 位置づけ文（例）: 「本ツールは、コスト上昇要因の公的データの提示と分野への紐付けを主目的とした支援ツールです。公的データに基づくグラフは客観的な事実として、価格協議の進め方・依頼文は『書き方の参考』としてご利用ください。」
- ガイド参考注記（例）: 「※ 価格協議の進め方は一般的な流れの参考です。特定の成果や成功を保証するものではありません。」
- 依頼文参考注記（例）: 「※ 依頼文は文面づくりの参考用テンプレートです。実績に基づくものではなく、自社の状況に合わせて必ず調整してください。」

### 18.2 データ取得レシピ

詳細は `docs/DATA_UPDATE.md` に集約。要点:
- CGPI: 日銀 REST API `getDataCode`（`db=PR01`、`code=PRCG20_…`、`startDate=201601` で10年）。CSV直返し。系列コードは `PRList.xlsx` で確定。
- 最低賃金: 厚生労働省「全国加重平均の推移」。
- 貿易統計: 財務省/e-Stat。HSコードの月次「金額÷数量＝輸入単価」。

### 18.3 指標カタログ

分野別の指標マッピング（品目名・系列コード/HSコード・出典・単位・頻度）は `docs/DATA_CATALOG.md` を単一の情報源とする。
