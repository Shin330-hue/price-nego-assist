# 要件定義書

**プロジェクト名:** 取適法・公的データ対応 価格協議準備ツール（金属加工業向け）  
**文書バージョン:** 2.0.0  
**作成日:** 2026-06-26  
**対象読者:** 開発者、CLI型AIコーディングエージェント、プロジェクトオーナー  
**想定公開形態:** X等で共有しやすい、ログイン不要の公開Webアプリ

---

## 1. プロジェクトの目的

### 1.1 背景

物価高騰、原材料費の上昇、エネルギーコストの上昇、人件費の上昇により、金属加工業を営む中小企業・小規模事業者は価格改定・価格転嫁の必要性に直面している。一方で、実際には以下の理由により価格協議に踏み出せない企業が多い。

- 取引先にどのように価格協議を申し入れればよいか分からない
- 「値上げ交渉」は取引関係を悪化させるのではないかという心理的ハードルがある
- 原材料費、電力費、ガス代、労務費等の上昇を客観的に示す資料を用意できない
- 取適法や価格転嫁に関する公的情報が存在しても、現場目線では探しづらく、使いづらい
- 自社の個別資料と公的資料をどう組み合わせて説明すればよいか分からない

本ツールは、金属加工業者が価格協議に入るための「最初の一歩」を支援する。個社の原価計算や法的判断を代行するのではなく、**公的データと取適法関連情報を整理し、価格協議に使える簡易資料を作成すること**を目的とする。

### 1.2 ツールの位置づけ

本ツールは、以下のように位置づける。

> 金属加工業の中小受託事業者が、取引先へ価格協議を申し入れる前に、公的なコスト上昇データ、取適法上の関係ポイント、協議の進め方、コピペ可能な依頼文を確認し、A4の価格協議準備シートとして出力できる公開Webツール。

本ツールは「価格を必ず上げるための交渉代行ツール」ではない。ユーザーが、法令の趣旨に沿った正当なルートで価格協議を始めるための準備ツールである。

## 2. 成功条件

### 2.1 プロダクトとしての成功条件

- 初めて訪問した金属加工業者が、3分以内に自社に近い加工分野を選び、関連する公的コスト指標を確認できる
- ユーザーが、公的データの出典、取得日、公表日、系列名、計算方法を確認できる
- ユーザーが、取適法に関する関係ポイントを、条文番号だけでなく実務的な言葉で理解できる
- ユーザーが、価格協議準備シートを印刷またはPDF保存できる
- ユーザーが、価格協議依頼文をコピーして取引先へのメール作成に使える
- ユーザー登録、ログイン、個人情報入力なしで利用できる
- X等で共有されたときに、目的が一目で伝わるOGP表示がされる

### 2.2 開発・品質としての成功条件

- `pnpm install`、`pnpm lint`、`pnpm test`、`pnpm build` が成功する
- 主要ブラウザの最新版で表示崩れがない
- 375px幅のスマートフォンでも主要機能が利用できる
- 取適法関連データは、すべて出典URL・確認日を持つ
- 公的統計データは、すべて出典URL・公表日・取得日・系列名・単位・基準年・計算メモを持つ
- 出典未確認のサンプルデータが混在する場合、UIに「サンプルデータ」警告が表示される
- 法的断定を避け、「個別判断は公的窓口・専門家へ」という注意書きを表示する

| 対象 | v1.0で外す理由 |

| リアルタイム統計取得 | 公的データの形式変更、CORS、出典確認の問題を避けるため |

## 5. 法令・公的情報の扱い方

### 5.1 法令情報の原則

- 取適法に関する情報は、公正取引委員会、e-Gov法令検索、政府広報、中小企業庁等の公的情報を一次または準一次の根拠とする
- 条文番号、制度名、禁止事項名は推測で書かない
- ツール上では法的断定を避ける
- 「この行為は違法です」と断定するのではなく、「問題となるおそれがあります」「確認してください」と表現する
- ユーザーが価格協議を求める権利や正当性を理解しやすくするが、個別案件の法的助言は行わない

### 5.2 v1.0で扱う主要な法令・指針ポイント

| ID | 表示名 | 根拠・位置づけ | UI上の説明方針 |
|---|---|---|---|
| LEGAL-001 | 取適法の目的・位置づけ | 発注者・受注者の対等な関係、価格転嫁・取引適正化 | 「価格協議を正当な手続きで進めるためのルール」と説明 |
| LEGAL-002 | 発注内容等の明示 | 取適法 第4条 | 取引条件を書面・電磁的方法で明らかにする重要性を説明 |
| LEGAL-003 | 代金の減額の禁止 | 取適法 第5条第1項第3号 | 協議後・発注後の一方的な減額リスクを説明 |
| LEGAL-004 | 買いたたきの禁止 | 取適法 第5条第1項第5号 | 著しく低い代金設定が問題となり得ることを説明 |
| LEGAL-005 | 協議に応じない一方的な代金決定の禁止 | 取適法 第5条第2項第4号 | 価格協議の求めに対し、実質的な協議なく一方的に代金を決めることが問題となり得ると説明 |
| LEGAL-006 | 報復措置の禁止 | 取適法 第5条第1項第7号 | 相談・申告・協議要請を理由に不利益取扱いをすることの問題を説明 |
| LEGAL-007 | 価格交渉の記録作成 | 労務費の適切な転嫁のための価格交渉に関する指針等 | 交渉日時、相手、提示資料、回答内容の記録を推奨 |

### 5.3 必須の注意書き

全画面のフッター、法令カード付近、印刷シート末尾に以下の趣旨を表示する。

> 本ツールは、価格協議の準備を支援するための参考情報を提供するものであり、法律相談・個別案件の適法性判断を行うものではありません。取適法の適用可否や個別の取引上の判断については、公正取引委員会、中小企業庁、取引かけこみ寺、価格転嫁サポート窓口、弁護士等の専門家にご確認ください。

## 6. 公的データの扱い方

### 6.1 データ取得方針

v1.0では、アプリ実行時に外部APIへアクセスしない。公的データは、リポジトリ内のJSONファイルとして保持する。各データは手動または更新スクリプトで更新し、出典メタ情報を必ず持たせる。

### 6.2 v1.0で扱うデータカテゴリ

| カテゴリ | 代表指標 | 想定出典 | 用途 |
|---|---|---|---|
| 原材料費 | 鉄鋼関連 企業物価指数 | 日本銀行 企業物価指数（2020年基準） | 鉄鋼材料価格上昇の公的根拠 |
| 原材料費 | 非鉄金属関連 企業物価指数 | 日本銀行 企業物価指数（2020年基準） | アルミ、銅、亜鉛等の価格上昇の公的根拠 |
| エネルギー費 | 電力関連 企業物価指数 | 日本銀行 企業物価指数（2020年基準） | 工作機械、レーザー、コンプレッサ等の電力負担説明 |
| エネルギー費 | 都市ガス・LPG等の価格指標 | 日本銀行 企業物価指数等 | 熱処理、鋳造、溶接等の熱源コスト説明 |
| 労務費 | 地域別最低賃金 全国加重平均 | 厚生労働省 | 労務費上昇の公的根拠 |
| 労務費 | 製造業賃金関連データ | 厚生労働省、e-Stat等 | 可能であれば追加。出典確認できない場合はv1.0では非表示 |

### 6.3 データ品質区分

| 区分 | 説明 | UI表示 |
|---|---|---|
| `official_verified` | 公的ソースから取得し、出典・公表日・取得日・系列名を確認済み | 通常表示 |
| `official_manual` | 公的ソースを手動転記し、確認日を記録済み | 通常表示。ただし手動更新であることを表示 |
| `sample` | 開発用サンプル値。公的値として利用不可 | 明確な警告を表示し、印刷シートにも警告を出す |
| `deprecated` | 古い基準年や更新停止データ | 原則非表示。必要な場合のみ警告付きで表示 |

### 6.4 表示上の注意

- 「基準年比」は、指数の基準値100と最新値を比較する場合のみ使う
- データ配列の先頭値と最新値の比較は「表示期間内の変化率」と表記する
- 前年比は、月次データなら12か月前、四半期データなら4期前、年次データなら1年前と比較する
- データ頻度が不明な場合、前年比を表示しない
- 小数点は原則1桁表示
- 参照した公的データは、値そのものだけでなく「交渉での使い方」を併記する

## 7. 機能要件詳細

### F-001 加工分野選択

#### 概要
トップページで、ユーザーが自社に近い加工分野を選択する。

#### 対象分野

| ID | 分野名 | 主なコスト要因 |
|---|---|---|
| `cutting` | 切削加工 | 鉄鋼・非鉄材料、電力、切削油・工具、労務費 |
| `welding` | 溶接・接合 | 鉄鋼・ステンレス、電力、ガス、溶接材料、労務費 |
| `grinding` | 研磨・表面処理 | 電力、砥石・研削液、労務費 |
| `pressing` | プレス加工 | 鋼板・アルミ板、電力、金型保全、労務費 |
| `sheet-metal` | 板金加工 | 鋼板・ステンレス、電力、ガス、労務費 |
| `plating` | めっき・コーティング | 非鉄金属、薬品、電力、水道・排水処理、労務費 |
| `casting` | 鋳造・鍛造 | 鉄鋼・アルミ、ガス・電力、砂・副資材、労務費 |
| `heat-treatment` | 熱処理 | ガス・電力、炉保全、労務費 |

#### 受け入れ条件

- 8分野がカードで表示される
- 各カードは、分野名、短い説明、主な影響コストを表示する
- カードクリックで `/fields/:fieldId` へ遷移する
- 不明な分野IDではトップページまたは404へ誘導する

---

### F-002 分野別コスト要因表示

#### 概要
選択分野において価格協議で説明しやすいコスト要因を表示する。

#### 表示項目

- 主要原材料
- エネルギー消費の特徴
- 労務費・技能要件
- 主要消耗品・副資材
- 価格協議で補足するとよい自社資料例

#### 受け入れ条件

- 分野ごとに内容が変わる
- 自社資料例は「任意で添付」とし、アップロードは求めない
- 「この公的データだけで個社の値上げ額を証明するものではない」という注意を出す

---

### F-003 公的コスト指標グラフ

#### 概要
分野に関連する公的指標を時系列グラフで表示する。

#### 表示項目

- 指標名
- 最新値
- 基準年比または表示期間内変化率
- 前年比。計算可能な場合のみ
- グラフ
- 出典名
- 出典URL
- 公表日
- 取得日
- 系列名
- 単位
- 基準年
- 計算メモ

#### 受け入れ条件

- 各指標は出典メタ情報を必ず持つ
- データ品質が `sample` の場合、画面上部と指標カードに警告を表示する
- グラフがスマートフォンでも横スクロールなしで読める
- グラフ下に「交渉での使い方」を表示する

---

### F-004 出典・真正性表示

#### 概要
ユーザーが取引先に説明できるよう、出典情報を明示する。

#### 表示方針

- 単に「日本銀行」ではなく、「日本銀行 企業物価指数（2020年基準） / 系列名：〇〇 / 公表対象月：〇〇 / 掲載日：〇〇」のように出す
- 最低賃金データは「厚生労働省 地域別最低賃金 全国一覧 / 全国加重平均」等のように出す
- 印刷シートにも出典一覧を出す

#### 受け入れ条件

- `sourceUrl`、`sourceName`、`publisher`、`retrievedAt`、`publishedAt`、`seriesName` が欠けているデータはビルドまたはテストで検出される

---

### F-005 取適法関連ポイント表示

#### 概要
価格協議に関係する取適法上のポイントを、実務的な言葉で表示する。

#### 表示方針

- 条文全文の長文表示ではなく、カード型の平易な説明にする
- 条文番号・タイトル・要旨・価格協議での意味・注意書き・出典を表示する
- 誤解を招くため「発注者は必ず値上げに応じなければならない」とは書かない
- 「協議に応じること」「必要な説明・情報提供」「一方的な代金決定を避けること」を中心にする

#### 受け入れ条件

- 法令ポイントはすべて `legal-bases.ts` または `legal-bases.json` に一元管理される
- 条文番号がハードコードで画面内に散らばらない
- 全法令カードに出典URLと確認日がある

---

### F-006 価格協議の進め方ガイド

#### 概要
価格協議の手順を、ユーザーがそのまま行動に移せる形で示す。

#### ガイドの標準ステップ

1. 対象取引を整理する
2. 影響しているコストを整理する
3. 公的データを確認する
4. 自社資料を手元に準備する。仕入先通知、電力明細、見積書、賃金改定資料など
5. 価格協議依頼文を送る
6. 協議日時・相手・提示資料・回答内容を記録する
7. 必要に応じて公的相談窓口へ相談する

#### 受け入れ条件

- ガイドは分野に関係なく共通表示される
- 分野別の補足がある場合はカード内に出す
- 相談窓口リンクを表示する

---

### F-007 価格協議準備シート生成

#### 概要
ユーザーが印刷・PDF保存して、社内検討や取引先への説明補助に使えるA4 1〜2枚の資料を生成する。

#### シート内容

- タイトル
- 作成日
- 選択分野
- この分野で影響しやすいコスト要因
- 参照した公的データの要約
- グラフまたはミニチャート
- 価格協議で伝えるポイント
- 取適法関連ポイント
- 自社で追記すべき資料チェックリスト
- 相談窓口・参考リンク
- 注意書き

#### 受け入れ条件

- ダッシュボードとは別に、白背景・ビジネス文書調の印刷レイアウトを持つ
- 印刷時、ナビゲーション、ボタン、装飾的背景を非表示にする
- 出典一覧が印刷される
- サンプルデータが含まれる場合、印刷シートにもサンプル警告が出る

---

### F-008 価格協議依頼文テンプレート

#### 概要
ユーザーがコピーして取引先へ送れる文面を表示する。

#### 文面方針

- 「価格改定を受け入れてください」ではなく「価格協議のお願い」とする
- 威圧的・断定的な表現を避ける
- 取適法は脅しではなく、協議を適切に進めるための背景として触れる
- 個別の上昇額・品番・改定希望日はユーザーが追記する空欄にする

#### 受け入れ条件

- コピー用ボタンを設置する
- 文面中に `[取引先名]`、`[対象品番・案件]`、`[希望協議日]` などのプレースホルダーを入れる
- 文字化けせずメールへ貼り付けられる

### F-009 印刷・PDF保存

#### 概要
ブラウザの印刷機能を使って、価格協議準備シートをPDF保存できる。

#### 受け入れ条件

- 「PDF保存・印刷」ボタンで `window.print()` を呼ぶ
- 印刷時はシート部分が中心になる
- 印刷レイアウトはA4縦を想定する
- 背景色は白、文字は黒・濃いグレーを基本とする

## 8. 非機能要件

### 8.1 技術・運用

| 項目 | 要件 |
|---|---|
| アーキテクチャ | 静的フロントエンドSPA |
| デプロイ先 | Vercel、Cloudflare Pages、GitHub Pagesのいずれか |
| パッケージ管理 | pnpm |
| 言語 | TypeScript |
| UI | React + Tailwind CSS |
| グラフ | Rechartsまたは同等の軽量グラフライブラリ |
| データ保存 | なし。ローカルストレージも原則使わない |
| 個人情報 | 入力・保存しない |
| 外部APIキー | フロントエンドに置かない |

### 8.2 パフォーマンス

- 初回ロードは標準的なブロードバンド環境で3秒以内を目指す
- 主要ページのLighthouse Performance 80以上を目指す
- グラフ表示は1秒以内を目指す
- 画像アセットは必要最小限にする

### 8.3 アクセシビリティ

- キーボード操作に対応する
- フォーカスリングを表示する
- 色だけに依存しない。上昇・注意・警告はテキストとアイコンを併用する
- コントラスト比に配慮する
- チャートの近くにテキスト要約を置く

### 8.4 セキュリティ

- APIキー、秘密情報、個人情報をフロントエンドに含めない
- `VITE_*_API_KEY` のようなクライアント露出キーは使わない
- 外部リンクは `noopener noreferrer` を付与する
- ユーザー入力はテンプレートのプレースホルダー程度に留め、保存しない

### 8.5 法務・表現

- 法律相談ではない旨を明示する
- 取適法適用の確定判定をしない
- 価格改定が必ず認められるような表現をしない
- 公的データは個社の値上げ額を直接証明するものではなく、交渉の客観的背景資料として位置づける

## 13. 追補（v1.1）：ツールの位置づけ・信頼区分・データ拡充方針

本節は v1.0 公開後の運用方針として追補する（本文の趣旨を変更するものではない）。

### 13.1 機能の優先度（位置づけ）

本ツールは「価格交渉を代行・保証する」ものではなく、**支援ツール**である。機能の優先度は次のとおり。

1. **コスト上昇要因の公的データの提供と、加工分野への紐付け**（中核）
2. データの見せ方（グラフ・出典の提示）
3. 文章（価格協議依頼文）の書き方の参考

### 13.2 信頼区分（重要）

ユーザーが各情報の確からしさを取り違えないよう、UI・印刷物で次を明確に区別する。

| 区分 | 位置づけ | 表示方針 |
|---|---|---|
| 公的データに基づくグラフ・数値 | 出典の明確な客観的事実。**そのまま根拠として利用してよい** | 肯定的な信頼キュー＋出典リンクを常時表示 |
| 価格協議の進め方ガイド | 一般的な流れの**参考** | 「参考」表示＋注記。特定の成果を保証しない |
| 価格協議依頼文テンプレート | 文面づくりの**参考**。実績に基づくものではない | 「参考」表示＋注記。自社状況に合わせ要調整 |

### 13.3 出典リンク方針

- すべての公的データ（指標カード・出典概要・出典一覧・印刷シート）に、控えめでも**出典リンクを必ず付す**。
- 外部リンクは別タブ＋`rel="noopener noreferrer"`。

### 13.4 データ拡充方針

- 公的指標は**過去10年（2016年〜）**を既定で表示する。
- 分野ごとに「自社の仕入れに近い品目」を見せるため、CGPIの**品目レベル**（特殊鋼切削工具・工具鋼・機械工具・溶接棒・研削と石・研磨材・めっき鋼板・ニッケル地金 等）まで拡充し、分野別にマッピングする（`docs/DATA_CATALOG.md`）。
- 工具材質など CGPI に無いものは**財務省 貿易統計**（タングステン等の輸入単価＝金額÷数量）で補う。取得・検証できたもののみ `official_manual`、不確実なものは `sample` として警告表示する。
